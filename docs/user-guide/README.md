# SkillForge AI — User Guides

Per-role one-pagers for day-one onboarding. Each one is a targeted ~5–12 minute read focused on one persona. For the comprehensive cross-role reference, see [`../USER_GUIDE.md`](../USER_GUIDE.md).

| Guide | Audience | Read time |
|---|---|---|
| [Employee](USER_GUIDE_EMPLOYEE.md) | Everyone — self-assessments + scorecards | ~8 min |
| [Manager](USER_GUIDE_MANAGER.md) | People managers scoring direct reports | ~10 min |
| [HR Admin](USER_GUIDE_HR_ADMIN.md) | Cycle owners, user management, exports | ~12 min |
| [AI Champion](USER_GUIDE_AI_CHAMPION.md) | Framework stewards + (Phase 2) calibration | ~6 min |
| [Leadership](USER_GUIDE_LEADERSHIP.md) | VP+, functional heads, read-only access | ~5 min |
| [Super Admin](USER_GUIDE_SUPER_ADMIN.md) | Tenant operators, break-glass, audit | ~10 min |

## How to use

- **For onboarding emails**: link directly to the role-specific guide.
- **For training**: walk through the guide matching the user's role; reach for the full guide only for edge cases.
- **For documentation updates**: edit the full guide first, then back-port changes to any affected one-pagers. The per-role files deliberately do not repeat everything — they summarize.

## Screenshots

Screenshot placeholders are embedded in each guide as `screenshots/<role>-<num>-<name>.png` references. See [`screenshots/README.md`](screenshots/README.md) for the capture brief (what each shot should show, what state the app should be in, what data to use).
